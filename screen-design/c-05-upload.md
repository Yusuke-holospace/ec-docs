## 2. 画面構成

### PC版

| セクション | 内容 |
|------------|------|
| ヘッダー（共通） | C-03と同じ |
| パンくずリスト | トップ > カート > データ入稿 |
| ステップインジケーター | カート → **データ入稿** → 注文確認 → 完了 |
| 商品サマリー | 入稿対象の商品情報 |
| アップロードセクション | スロット別アップロード＋入稿要件＋テンプレートDL |
| アクションボタン | 「カートに戻る」「入稿を完了してカートに戻る」 |
| パンくずリスト（下部） | フッター直上 |
| フッター（共通） | C-03と同じ |

### SP版

| セクション | 内容 |
|------------|------|
| ヘッダー（共通） | C-03 SP版と同じ |
| ステップインジケーター | コンパクト表示 |
| 商品サマリー | PC版と同じ |
| アップロードセクション | ドラッグ&ドロップなし。「ファイルを選択」ボタンのみ |

**SP版でのPC版との主な差分：**
- パンくず非表示

- ボタン縦並び

## 3. セクション詳細

### 3.1 ステップインジケーター

| ステップ | 状態 |
|----------|------|
| 1. カート | 完了（緑） |
| 2. データ入稿 | 現在（黒・アクティブ） |
| 3. 注文確認 | 未到達（グレー） |
| 4. 完了 | 未到達（グレー） |

### 3.2 商品サマリー

| 要素 | 詳細 | 設定元 |
|------|------|--------|
| 商品画像 | サムネイル | 商品別 |
| 注文種別バッジ | 「通常注文」/「サンプル制作」 | カートデータ |
| 商品名 | 商品の正式名称 | 商品別 |
| オプション | 選択済みオプション＋数量を表示 | カートデータ |
| 金額 | この商品の小計 | 自動算出 |

### 3.3 アップロードセクション

#### 入稿スロット

管理者が商品ごとに入稿スロット（必要なファイルの種類）を定義。1スロット＝1ファイル。スロットは商品に対して常に表示されるものと、特定のオプション選択時にのみ表示されるものがある。

| 要素 | 詳細 | 設定元 |
|------|------|--------|
| スロットラベル | 「表面デザイン」「裏面デザイン」「台紙デザイン」等 | 商品別（管理者が定義） |
| 必須/任意 | スロットごとに設定。必須は赤字「必須」、任意はグレー「任意」 | 商品別 |
| 表示条件 | 常に表示、または特定オプション選択時のみ表示（例: 台紙印刷オプション「あり」の場合に台紙デザインスロットが出現） | 商品別（管理者がオプションとスロットの紐づけを定義） |
| アップロードエリア（PC） | ドラッグ&ドロップ＋「ファイルを選択」ボタン | — |
| アップロードボタン（SP） | 「ファイルを選択」ボタンのみ（ドラッグ&ドロップなし） | — |
| アップロード済み状態 | アップロードエリアがファイル情報（アイコン＋ファイル名＋サイズ＋「変更」リンク）に切り替わる | — |

#### 入稿データの要件

| 要素 | 詳細 | 設定元 |
|------|------|--------|
| 要件リスト | 対応形式、カラーモード、解像度、注意事項、ファイルサイズ上限等 | 商品別（商品詳細ページの入稿仕様と同じデータ） |

#### テンプレートダウンロード

| 要素 | 詳細 | 設定元 |
|------|------|--------|
| テンプレートDL | 選択中のオプションに対応するテンプレートファイルをダウンロード | 商品別 |

### 3.4 アクションボタン

| 要素 | 詳細 |
|------|------|
| カートに戻る | C-04（カートページ）へ遷移。入稿データは保存されない |
| 入稿を完了してカートに戻る | 必須スロットが全てアップロード済みの場合のみ活性。クリックでデータを保存しC-04へ遷移。入稿状態が「入稿済み」に更新される |
| エラーメッセージ | 必須スロット未入稿時、ボタン下に「必須の入稿データが不足しています」と表示 |

## 4. インタラクション

| アクション | 動作 |
|------------|------|
| ファイル選択 / ドラッグ&ドロップ（PC） | ファイルをバリデーション後、アップロード。成功するとアップロードエリアがファイル情報に切り替わる |
| ファイル選択（SP） | デバイスのファイルピッカーが開く。選択後アップロード |
| ファイル「変更」クリック | アップロード済みファイルを別ファイルに差し替え |
| 「入稿を完了してカートに戻る」 | データを保存しC-04へ遷移。カートの入稿状態が「入稿済み」に更新 |
| 「カートに戻る」 | C-04へ遷移。入稿データは保存されない |

---

## 5. バリデーション・エラー

| 条件 | 表示 |
|------|------|
| 対応していないファイル形式 | スロット内にエラーメッセージ「対応していないファイル形式です。AI, PSD, PDF形式でアップロードしてください。」。スロット枠線が赤に |
| ファイルサイズ超過 | スロット内にエラーメッセージ「ファイルサイズが上限（200MB）を超えています。」 |
| 必須スロット未入稿 | 「入稿を完了」ボタンを非活性化。ボタン下にエラーメッセージ |
| アップロードエラー（通信等） | スロット内にエラーメッセージ「アップロードに失敗しました。再度お試しください。」 |

---

## 6. 補足事項

| 項目 | 内容 |
|------|------|
| アップロード中のプログレス | 大きなファイル（200MBまで対応）ではプログレスバーを表示。実装時にエンジニアと相談 |
| ファイルプレビュー | 初期リリースではファイル名＋サイズのみ。将来的にPDFのサムネイルプレビュー対応を検討 |
| 再入稿モード | カートの「データを変更」から遷移した場合、既存のアップロード済みファイルが表示された状態で開く |

---

## 7. データ要件

| データ | 内容 | 設定元 |
|--------|------|--------|
| 商品情報 | 商品名、画像、オプション、数量、金額 | カートデータ＋商品別 |
| 入稿スロット定義 | スロットのラベル、必須/任意、受け付けるファイル形式、表示条件（常時 or 特定オプション選択時） | 商品別（管理者が商品ごとに定義。オプションとの紐づけを含む） |
| 入稿データの要件 | 対応形式、カラーモード、解像度、注意事項等 | 商品別（商品詳細ページの入稿仕様と同じ） |
| 入稿テンプレート | スロットごとのテンプレートファイル。スロットと同様にオプション選択に連動して表示が変わる | 商品別 |
| アップロード済みファイル | 再入稿時に既存データを表示 | サーバー |
