# 画面設計書：C-12 注文履歴一覧ページ

> ワイヤーフレーム PC: `wireframes/c-12-orders.html` — [プレビュー](https://yusuke-holospace.github.io/ec-docs/wireframes/c-12-orders.html)
> ワイヤーフレーム SP: `wireframes/c-12-orders-sp.html` — [プレビュー](https://yusuke-holospace.github.io/ec-docs/wireframes/c-12-orders-sp.html)

## 1. 画面概要

| 項目 | 内容 |
|------|------|
| 画面ID | C-12 |
| 画面名 | 注文履歴一覧ページ |
| URL | `/mypage/orders` |
| 認証 | 必要 |
| 対応デバイス | PC・スマートフォン（レスポンシブ） |

---

## 2. 画面構成

### PC版

| セクション | 内容 |
|------------|------|
| ヘッダー（共通） | C-03と同じ |
| パンくずリスト | トップ > マイページ > 注文履歴 |
| サイドナビ＋コンテンツ | 2カラム。サイドナビで「注文履歴」がアクティブ |
| フィルター | ステータス・種別による絞り込み |
| 注文カード一覧 | 注文ごとのカード |
| パンくずリスト（下部） | フッター直上 |
| フッター（共通） | C-03と同じ |

### SP版

| セクション | 内容 |
|------------|------|
| ヘッダー（共通） | C-03 SP版と同じ |
| 「マイページに戻る」バー | サイドナビの代わり |
| フィルター | 横スクロール |
| 注文カード一覧 | PC版と同じ |
| フッター（共通） | C-03 SP版と同じ |

**SP版でのPC版との主な差分：**
- サイドナビ → 「マイページに戻る」バー
- パンくず非表示
- フィルターボタンが横スクロール

---

## 3. セクション詳細

### 3.1 フィルター

| フィルター | 内容 |
|------------|------|
| すべて | 全注文を表示（デフォルト） |
| 通常注文 | 通常注文のみ |
| サンプル制作 | サンプル制作のみ |
| 進行中 | 完了以外のステータス |
| 完了 | 完了ステータスのみ |

### 3.2 注文カード

| 要素 | 詳細 | 設定元 |
|------|------|--------|
| 注文番号 | ORD-XXXXXXXX-XXXX | サーバー |
| 注文日 | YYYY/MM/DD | サーバー |
| ステータスバッジ | 現在のステータスを色付きバッジで表示（下記参照） | サーバー |
| 商品概要 | 商品名＋数量の簡潔な一覧（例: 缶バッジ 44mm x100 / アクスタ 100mm x200 / 他1点） | サーバー |
| 注文種別 | 「通常注文」「サンプル制作」「通常＋サンプル」 | サーバー |
| 合計金額 | 税込 | サーバー |
| クリック | C-13（注文詳細）へ遷移 | — |

**ステータスバッジの色定義：**

| ステータス | 色 |
|------------|-----|
| 入稿受付 | 青 |
| 入稿確認中 | 青 |
| 差し戻し | 赤 |
| 承認済み | 青 |
| 決済待ち | オレンジ |
| 入金確認 | 青 |
| サンプル受付 | 青 |
| 製造中 | 青 |
| 発送済み | 青 |
| 完了 | 緑 |

### 3.3 ページネーション

| 要素 | 詳細 |
|------|------|
| 表示件数 | 1ページあたり20件 |
| PC | ページ番号ナビゲーション（1, 2, 3... 次へ） |
| SP | 「もっと読み込む」ボタン（追加読み込み方式） |

---

## 4. インタラクション

| アクション | 動作 |
|------------|------|
| フィルタークリック | 選択したフィルターで一覧を絞り込み。アクティブなフィルターは黒背景に |
| 注文カードクリック | C-13（注文詳細）へ遷移 |
| ページネーション（PC） | 該当ページの注文を表示 |
| 「もっと読み込む」（SP） | 次の20件を追加読み込み |

---

## 5. 補足

- 注文が0件の場合は「注文履歴がありません」＋「商品を探す」ボタンを表示
- 差し戻しステータスの注文は目立つように赤バッジで表示し、購入者が対応に気づきやすくする
