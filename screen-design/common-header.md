# 画面設計書：共通ヘッダー

全購入者向けページ（C-01〜C-20）で共通のヘッダー構成。
C-08（ログイン）/ C-09（会員登録）/ C-21（共有ステータス）はミニマルヘッダー（ロゴのみ）のため対象外。

## 3. スクロール時の挙動

| 状態 | PC | SP |
|------|----|----|
| 初期表示 | 告知バー + 1段目 + 2段目 | 1段目 + 2段目（告知バーなし） |
| スクロール後 | **1段目のみ sticky 固定**。告知バー・カテゴリナビは流れて消える | **1段目 + 2段目（検索バー）を sticky 固定** |

---

## 4. 画像のホバー切り替え

カテゴリ画像・商品サムネイルはホバー時に別画像に切り替わる。

| 項目 | 値 |
|------|-----|
| 通常状態 | ①画像を表示（img-default） |
| ホバー状態 | ②画像を表示（img-hover） |
| トランジション | opacity 0.3s |
| 対象箇所 | カテゴリカード（C-01）、メガメニュー内サムネイル |

## 6. アイコンライブラリ

| 項目 | 詳細 |
|------|------|
| ライブラリ | **Lucide** |
| パッケージ | `lucide-react`（実装時）|
| 理由 | Next.js（React）との相性が良い。Tree-shaking対応で軽量。シンプルなデザインがサイトのトーンに合う |
| CDN（ワイヤーフレーム用） | `https://unpkg.com/lucide@latest` |
| 初期化（ワイヤーフレーム用） | `lucide.createIcons()` をページ末尾で呼び出し |
| アイコン一覧 | https://lucide.dev/icons |

---

## 7. 実装上の注意

- ヘッダーは全ページ共通コンポーネントとして実装する（React: `<Header />` コンポーネント）
- ログイン状態の判定は認証コンテキスト（Supabase Auth）から取得
- カートバッジの数字はカート状態（ローカルストレージ or サーバー）から取得
- 検索はEnterキーまたはアイコンクリックで実行。`/products?q={キーワード}` に遷移
- メガメニューのデータ（カテゴリ・サブ商品）は商品マスタから動的に取得
- SP版のハンバーガーメニュー展開時の内容（カテゴリ一覧・ご利用ガイド・FAQ等）は別途定義が必要
