## 2. セクション構成

| セクション | 内容 |
|------------|------|
| 注文ヘッダー＋操作ボタン | 注文番号・ステータス・チャネル・顧客名＋承認/差し戻し/決済案内/ステータス変更 |
| ステータス履歴 | タイムライン形式 |
| 連絡先 | 注文時に指定されたメール・電話（会員登録情報と異なる場合あり） |
| 注文商品＋入稿データ | 商品一覧＋各商品の入稿ファイルプレビュー・ダウンロード |
| お支払い | 支払い方法・商品合計・送料・納期目安・合計 |
| 配送先 | 配送先住所 |
| 担当スタッフ | プルダウンで割り当て |
| 仕上がりイメージ | 画像アップロード（顧客にメール通知） |
| 共有ステータスURL | C-21のURL表示＋コピー |
| 同意記録 | 同意日時・同意項目 |

## 3. セクション詳細

### 3.1 注文ヘッダー＋操作ボタン

| 要素 | 詳細 | 設定元 |
|------|------|--------|
| 注文番号 | ORD-XXXXXXXX-XXXX | サーバー |
| ステータスバッジ | 現在のステータス（色付き） | サーバー |
| チャネルバッジ | EC / 電話 / メール / 営業 | サーバー |
| 注文日時 | YYYY/MM/DD HH:MM | サーバー |
| 顧客名 | お名前（会社名） | サーバー |

#### 操作ボタン（ステータスに応じて活性/非活性）

| ボタン | 動作 | 活性条件 |
|--------|------|---------|
| 承認 | 入稿データを承認。ステータスを「承認済み」に更新。NTF-003を送信 | 入稿確認中 |
| 差し戻し | 差し戻しメール（NTF-004）を送信。差し戻し理由の入力モーダルを表示 | 入稿確認中 |
| 決済案内を送信 | 支払い方法に応じた決済案内メール（NTF-005）を送信 | 承認済み |
| ステータス変更 | プルダウンでステータスを選択して更新。NTF-011を自動送信 | 全ステータス |

### 3.2 ステータス履歴

| 要素 | 詳細 |
|------|------|
| 表示形式 | 縦タイムライン。C-13と同じ形式 |
| 完了ステップ | 緑ドット＋日時表示 |
| 現在ステップ | 青ドット |
| 未到達ステップ | グレードット |

### 3.3 連絡先

| 要素 | 詳細 | 設定元 |
|------|------|--------|
| メールアドレス | 注文時にC-06で指定された連絡先 | サーバー |
| 電話番号 | 同上 | サーバー |
| 注記 | 「注文時に指定された連絡先（会員登録情報と異なる場合があります）」 | 固定テキスト |

### 3.4 注文商品＋入稿データ

C-13と同じ商品一覧に加え、各商品の入稿データを表示。

| 要素 | 詳細 | 設定元 |
|------|------|--------|
| 商品情報 | 画像・種別バッジ・商品名・オプション・数量 | サーバー |
| 金額 | 小計（単価×数量＋固定費） | サーバー |
| 入稿ステータス | 「入稿済み」（緑）/ 「要確認」（赤） | サーバー |
| 入稿ファイル | ファイル名・サイズ・形式アイコン | サーバー |
| プレビューリンク | クリックでファイル内容をプレビュー表示 | — |
| ダウンロードリンク | クリックでファイルをダウンロード | — |
| 要確認の商品 | 背景色を薄赤（#fff8f8）で強調 | — |

### 3.5 お支払い

| 要素 | 詳細 | 設定元 |
|------|------|--------|
| 支払い方法 | クレジットカード / 銀行振込 / 請求書払い | サーバー |
| 商品合計（N点） | 全商品の小計合計 | サーバー |
| 送料 | 無料 or 金額 | サーバー |
| 納期目安 | 約X営業日（入金確認完了後から起算） | サーバー |
| 合計（税込） | 商品合計＋送料 | サーバー |

### 3.6 配送先

| 要素 | 詳細 | 設定元 |
|------|------|--------|
| 会社名 | 任意（未入力の場合は非表示） | サーバー |
| お名前 | — | サーバー |
| 住所 | 郵便番号＋住所 | サーバー |
| 電話番号 | — | サーバー |

### 3.7 担当スタッフ（ADM-014）

| 要素 | 詳細 |
|------|------|
| プルダウン | 管理者スタッフ一覧から選択。「未割り当て」がデフォルト |
| 保存ボタン | 選択後に保存 |
| 表示先 | C-13（注文詳細）に担当者名として表示 |

### 3.8 仕上がりイメージ（ADM-013）

| 要素 | 詳細 |
|------|------|
| 画像アップロード | 複数枚アップロード可能。背景透過PNG等にも対応 |
| 自動通知 | アップロード時にNTF-012を顧客に自動送信 |
| 表示先 | C-13・C-21から一括ダウンロード（ZIP）可能 |

### 3.9 共有ステータスURL

| 要素 | 詳細 |
|------|------|
| URL表示 | C-21のトークン付きURLを表示（読み取り専用） |
| コピーボタン | クリックでクリップボードにコピー |
| 用途 | EC注文・代理注文共通。ログイン不要でステータス確認可能 |

### 3.10 同意記録

| 要素 | 詳細 | 設定元 |
|------|------|--------|
| 同意日時 | 注文確定時の日時 | サーバー |
| 同意項目 | 同意した項目の一覧 | サーバー |

## 4. インタラクション

| アクション | 動作 |
|------------|------|
| 「承認」クリック | 確認ダイアログ → 承認処理 → ステータス更新＋NTF-003送信 |
| 「差し戻し」クリック | 差し戻し理由入力モーダル → 送信 → ステータス更新＋NTF-004送信 |
| 「決済案内を送信」クリック | 支払い方法に応じた決済案内メール（NTF-005）を送信 |
| 「ステータス変更」クリック | プルダウン表示 → 選択 → 更新＋NTF-011送信 |
| 入稿ファイル「プレビュー」クリック | モーダルでファイルプレビュー表示 |
| 入稿ファイル「ダウンロード」クリック | ファイルをダウンロード |
| 担当スタッフ選択→「保存」 | 担当者を保存 |
| 仕上がりイメージ「追加」クリック | ファイルピッカー → アップロード → NTF-012自動送信 |
| 共有URL「コピー」クリック | URLをクリップボードにコピー |
