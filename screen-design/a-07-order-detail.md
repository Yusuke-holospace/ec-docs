## 3. セクション詳細

### 3.1 注文ヘッダー

| 要素 | 詳細 | 設定元 |
|------|------|--------|
| 注文番号 | ORD-XXXXXXXX-XXXX | サーバー |
| チャネルバッジ | EC / 電話 / メール / 営業 | サーバー |
| 注文日時 | YYYY/MM/DD HH:MM | サーバー |
| 顧客名 | お名前（会社名） | サーバー |

> ステータスは商品単位で管理する。注文単位のステータスは持たず、各商品行のステータスバッジ＋プルダウンで確認・変更する（3.3 注文商品＋入稿データ参照）。
> 決済案内は管理画面外で手動送信（システム対象外）。

### 3.2 連絡先

| 要素 | 詳細 | 設定元 |
|------|------|--------|
| メールアドレス | 注文時にC-06で指定された連絡先 | サーバー |
| 電話番号 | 同上 | サーバー |
| 注記 | 「注文時に指定された連絡先（会員登録情報と異なる場合があります）」 | 固定テキスト |

### 3.3 注文商品＋入稿データ

C-13と同じ商品一覧に加え、各商品の入稿データと**商品ごとのステータス変更UI**を表示。

| 要素 | 詳細 | 設定元 |
|------|------|--------|
| 商品情報 | 画像・種別バッジ・商品名・オプション・数量 | サーバー |
| 金額 | 小計（単価×数量＋固定費） | サーバー |
| 商品ステータスバッジ | 商品ごとの現在ステータス（色付き） | サーバー |
| 商品ステータスプルダウン | 商品ごとに常時表示。選択変更時の挙動はステータスにより分岐（下記参照） | — |
| 入稿ファイル | ファイル名・サイズ・形式アイコン | サーバー |
| プレビューリンク | クリックでファイル内容をプレビュー表示 | — |
| ダウンロードリンク | クリックでファイルをダウンロード | — |
| 差し戻し時の表示 | ステータスが「差し戻し」の商品は背景が薄赤（#fff8f8）になり、差し戻し理由を表示 | サーバー |

**商品ステータスプルダウン変更時の挙動:**

| 選択ステータス | 挙動 |
|----------------|------|
| 承認済み | 確認ダイアログ → 確認で更新＋**NTF-003**（承認通知）を送信 |
| 差し戻し | **差し戻し理由入力モーダル**を表示 → 送信で更新＋**NTF-004**（差し戻し通知）を送信。該当商品の背景が薄赤になり差し戻し理由を表示 |
| 製造中/発送済み/完了 | 確認ダイアログ「この商品のステータスを変更しますか？（NTF-011が送信されます）」→ 確認で更新＋**NTF-011**を送信（対象商品名と新ステータスをメール本文に記載） |
| 決済待ち / 入金確認 | プルダウンには表示されない（**全商品同期遷移**のため、商品単位の選択不可）。決済待ちは決済案内送信時に、入金確認は決済確認時に**注文内の全商品を一斉に**システムが自動遷移する |

| 要素 | 詳細 |

### 3.7 仕上がりイメージ（ADM-013）

| 要素 | 詳細 |
|------|------|
| 画像アップロード | 複数枚アップロード可能。背景透過PNG等にも対応 |
| 画像削除 | 各画像の×ボタンで個別削除 |
| 一括ダウンロード（ZIP） | 管理者もA-07上からZIP形式で一括ダウンロード可能 |
| 自動通知 | アップロード時にNTF-012を顧客に自動送信 |
| 表示先 | C-13・C-21からも一括ダウンロード（ZIP）可能 |

### 3.8 共有ステータスURL

| 要素 | 詳細 |
|------|------|
| URL表示 | C-21のトークン付きURL（`/orders/status/[token]`）を表示（読み取り専用） |
| コピーボタン | クリックでクリップボードにコピー |
| 用途 | 顧客・取引先と共有してログイン不要でステータス確認可能。EC注文・代理注文共通 |

> 運営内部（社内Slack等）で**この管理画面（A-07）自体を共有**したい場合は、ページヘッダー右上の「共有」ボタンから現在の管理画面URL（`/admin/orders/[id]`）をコピーできる。共有先は管理者ログインが必要。

### 3.9 管理者コメント

社内用のコメント。履歴形式で複数件記録可能。顧客には表示されない。**A-09会員詳細でも同じ仕様**。

| 要素 | 詳細 |
|------|------|
| コメントエントリ | 各コメントは **日付・記入者・本文** の3項目で構成。新しい順に表示 |
| 新規投稿 | テキストエリア＋「投稿」ボタン。投稿時に日付（自動）と記入者（ログイン中の管理者名・自動）が記録される |
| 最大文字数 | **1000文字**。超過時は入力欄のカウンタが赤になり、投稿ボタン非活性化 |
| ページネーション | **直近10件**を表示 + 「もっと見る」ボタンで追加10件ずつ読み込み |
| 削除 | **投稿者本人のみ**削除可。確認ダイアログ後に削除 |
| 編集 | 不可（記録性を担保） |
| 退職した管理者のコメント削除 | 現状は不可（投稿者本人のみ）。将来管理者ロールが細分化された際に上位ロールが削除可能になる予定 |
| 削除監査ログ | 初期スコープ外（必要に応じて将来対応） |
| メンション機能 | 初期スコープ外 |

## 4. インタラクション

| アクション | 動作 |
|------------|------|
| 商品ステータスプルダウンで「承認済み」選択 | 確認ダイアログ → 確認で該当商品のステータスを「承認済み」に更新＋**NTF-003**送信 |
| 商品ステータスプルダウンで「差し戻し」選択 | 差し戻し理由入力モーダル → 送信で該当商品のステータスを「差し戻し」に更新＋**NTF-004**送信。C-13で該当商品に「再入稿する」ボタンが表示される |
| 商品ステータスプルダウンでその他ステータス選択 | 確認ダイアログ → 確認で該当商品のステータスを更新＋**NTF-011**送信（対象商品名と新ステータスをメール本文に記載） |
| 入稿ファイル「プレビュー」クリック | モーダルでファイルプレビュー表示 |
| 入稿ファイル「ダウンロード」クリック | ファイルをダウンロード |
| 担当スタッフ選択→「保存」 | 担当者を保存 |
| 仕上がりイメージ「追加」クリック | ファイルピッカー → アップロード → NTF-012自動送信 |
| 共有URL「コピー」クリック | URLをクリップボードにコピー |
| 管理者コメント「投稿」クリック | テキストエリアの内容を投稿。日付・記入者を自動記録し一覧に追加（最新が一番上） |
| 管理者コメント「削除」クリック | 自分が投稿したコメントのみ表示。確認ダイアログ後に削除 |
